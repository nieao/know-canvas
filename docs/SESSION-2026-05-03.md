## 轮次 1 · 节点组团重叠修复

**用户反馈**: "以后生成的节点组团跑的特别远, 有些问题还有, 你再检查一下不同的组团生成以后的重叠问题, 把重叠问题再修正一下, 修正之前先 git 一下"

### 根因 (3 个)

1. **collision 假阴性** — `getNodeAbsoluteBox` 对所有 `node.type === 'group'` 一律 fallback `800×600`, 但实际:
   - projectGroup `1600×1100`
   - decomposeGroup `1260×260`
   - challengeGroup `380×800`

2. **跑得远** — `findFreePosition` `MAX_TRIES=50, STEP=80` → 累积 `4000px` 偏移, 找不到位置时返回最远坐标, 节点出视野.

3. **拆解组重叠** — `decomposeOntologyFurther` 用 `existingDgs.length × 偏移` 计算新 dg 的 y. 中间删一个 dg 后, 新 dg 与剩下的撞.

### 修复 (commit `e35484f`)

```javascript
// A 修 — 按子类型给精确 fallback
if (node.data?.isChallengeGroup)  { fallbackW = 380;  fallbackH = 800 }
else if (node.data?.isDecomposeGroup) { fallbackW = 1260; fallbackH = 260 }
else if (node.data?.isProjectGroup)   { fallbackW = 1600; fallbackH = 1100 }
else if (node.data?.isOrphanFallback) { fallbackW = 0;    fallbackH = 0 }

// B 修 — 限偏移上限
const MAX_TRIES = 16, STEP = 100, FALLBACK_TRY = 4
// 找不到时回退到第 4 次 (~400px), 不取最远

// C 修 — 用真实 bbox 而不是 length
let baseY = groupY + groupH + 60
for (const dg of existingDgs) {
  const box = getNodeAbsoluteBox(dg, nodes)
  baseY = Math.max(baseY, box.y + box.h + 40)
}
```

## 关键学习与沉淀

### 用户反馈 → 触发的元规则

1. "git 一个版本" / "修正之前先 git 一下" → 风险操作前必须先 commit 当前状态
2. "都是名字 (分工) 吧" → 称呼一律 `lichang (hermes 框架策划) · 小叶子 (产品策划) · 你想猫 (架构策划)`, 顺序固定
3. "红圈写成一排可以" → CSS layout 改动后用 playwright 截图验证再 commit
4. "你自查一下" → 三道闸: canvas-render / collab-sync / design-audit
5. "不要覆盖原 ppt" → 风险新增不破坏现有, 用副本 + 副名

### 不要做什么 (negative space)

- 看到 React Flow 002 不要重新发明轮子, 直接读 `domain-knowledge.md#bug-1` 套 v4 cascade hidden + filter
- `findFreePosition` 找不到位置时不要返回最远坐标, 必须用 `FALLBACK_TRY=4` 回退到 ~400px
- 给 group 不要一律 fallback `800×600`, 必须按 `isProjectGroup / isDecomposeGroup / isChallengeGroup / isOrphanFallback` 分支
- 不要每页手写 slide-credit, 用 JS `slides.forEach` 自动注入
