// 01 / WHAT IS IT

三句话讲清, 它能干啥

你有一个想法, 比如 "产品该不该上线灰度", 或者 "我要跳槽吗"。 你想得头头是道, 但其实有 N 个隐含假设你没验过 — Aletheia 让 AI 来当杠精, 一个一个戳穿你, 然后给你具体待办。

01

写下你的想法

把你想做的事情, 用一两句话写在画布上 — 不需要写得很完整, 系统会自动帮你拆成几个核心提议节点。

// concept · task · ontology 都行
02

启动决策引擎

点右下角 ALETHEIA 按钮, 选择场景 (ToB/ToC/ToG), AI 杠精 / 审计师 / 苏格拉底三个角色会同时上场, 一条一条产出反驳节点。

// 每条反驳含论据 + 具体待办
03

看 Action Plan

所有反驳跑完后, 会自动生成一个综合节点 (Synthesis), 含 HealthScore + 3-5 条优先级行动 + 风险清单。

// 03 / USER FLOW

从想法到行动清单, 4 步

没有学习成本。打开画布 → 写一句话 → 启动引擎 → 看 Action Plan。整个过程 30 秒~2 分钟, 全程不用切窗口。

STEP 01
进入画布

填个用户名, 点 "快速进入主房间"。多人协作自动同步, 同一房间所有人实时看到节点变化。

STEP 02
写下想法

左下角输入框写一句话目标 → AI 自动拆解成本体框架 (条件/路径/约束)。也可以直接双击画布建概念节点。

STEP 03
启动引擎

右下角 ALETHEIA 按钮 → 选场景 (ToB/ToC/ToG) → 选反驳人格 (Reddit/审计/苏格拉底) → 点 "对画布跑一轮"。

STEP 04
看清单

反驳节点逐个长出来 (一条 0.6 秒) → 综合节点出现 → 点 "查看完整方案" 看 P0/P1/P2 行动 + owner + 截止。

// 04 / PERSONAS

三个 AI 角色, 三种戳穿方式

不同角色的口吻和切入点不一样。Reddit 杠精是直球攻击, 审计师追合规和数字, 苏格拉底问根本前提。同一个想法跑三个人格, 漏洞曝光更全面。

⚔
REDDIT MODE

Reddit 杠精

"这个前提你默认成立, 但根本没人验过 — 举个反例就崩。"

直球质疑前提、攻击未验证假设, 适合早期想法压力测试。

⚖
AUDIT MODE

审计师

"数据隐私边界未明确, 10x 用户量时合规风险指数级上升。"

关注合规、ROI、量化数据, 适合上线前的最终把关。

🜨
SOCRATIC MODE

苏格拉底

"如果用户量减半, 这个结论还成立吗? 边界在哪?"

追问根本假设和边界条件, 适合战略层面的概念澄清。

// 05 / SCENARIOS

三种场景, 三套权重

ToB 关注合同/合规/SLA, ToC 关注用户体验/留存/CAC, ToG 关注政策/审批/数据安全。引擎按场景自动调整反驳的关注点。

SCENARIO A

ToB · 企业服务

关注 合同条款 / SLA / 合规边界 / 客户结构。
反驳重点: 客户合同里的隐藏条款 / 续约风险 / 服务可靠性证据.

SCENARIO B

ToC · 消费者

关注 用户体验 / 留存 / CAC/LTV / 增长路径。
反驳重点: 转化漏斗哪一步会掉 / 留存假设是否成立 / 单位经济模型.

SCENARIO C

ToG · 政府/监管

关注 政策合规 / 审批路径 / 数据安全 / 公开性。
反驳重点: 审批链条的不确定性 / 数据跨境 / 招投标条件.
