## 📋 现场演示三段式（建议讲述顺序）

### 段 1 · 多人共享画布（30 秒）

1. 三人同时打开演示站（同一房间号）
2. 强调右上角的 **三个用户头像**（不同颜色圆点 = 不同协作者）
3. 一人拖一个节点，其他屏幕上看到 **暖色脉冲气泡 + 拖动轨迹**
4. 一句话：「黑客松通过 Yjs CRDT，让三人画布零延迟同步，离线后回来自动合并」

### 段 2 · 三人共享项目库（30 秒）

1. 任一人点右上角 **「项目库」**按钮
2. 强调顶部 **「按用户筛选」chip 行**：能看到三个 owner 头像 + 项目数
3. 卡片角落每个项目带 **owner 圆点 + 用户名**
4. 一句话：「项目库本地 + yjs 双写，每个项目的创建者贴在卡片上，三人都能看到、都能载入」

### 段 3 · 时间轴回放（最亮点 · 30 秒）

1. 找一个有 **「N 帧」标签** 的项目卡片，点 **「▶ 回放」**
2. 项目库自动关闭，画布外圈出现 **暖色脉冲发光边框 + 顶部 PLAYBACK 标签**
3. 底部出现 **scrubber 控制条**：
   - 拖动滑块切到不同帧（拆解前 → 反驳后 → 综合完成）
   - 左上角 **「按项目 / 按用户」**切换
   - 切到「按用户」点不同 owner chip → 仅看这个用户的活动帧
4. 点 **退出** 回到协作画布
5. 一句话：「每一次拆解、反驳、综合都自动留快照，可以按时间或按用户回放整个决策过程」

## 🚀 暖场操作（演示前 5 分钟在自己机器上做好）

为了 demo 时回放有内容，演示前先做一遍：

1. 进入 demo 房间，先**手动建几个节点**
2. 点 **「保存」** 按钮 → JSON 保存（这会触发自动归档项目）
3. 改一些节点（拖动 / 加节点 / 改字）
4. 再点 **「保存」** → 项目库里同名项目的 commits 数 +1
5. 重复 3-4 次 → commits 累到 5+ 帧 → 回放才丰富

> **关键提示**：commits 是在「同一项目 id update」时累积的。如果每次都点「自动归档」生成新项目，每个项目只有 1 帧。所以 demo 前应该用 **「保存为 JSON」**或**重命名同一项目**多次保存。

## 🎯 评委可能问的问题 + 答案

| 问 | 答 |
|----|----|
| 怎么解决多人编辑冲突？ | Yjs CRDT，无中心服务器编排，本地操作即时生效，远端自动合并 |
| 回放数据存哪？ | yjs `projects` map，每个项目 commits[] 限 30 帧，自动滚动淘汰旧帧 |
| 这和普通的协作画布有什么不同？ | ALETHEIA 是「逻辑对抗决策引擎」，节点不只是图，还有元认知/反驳/综合 5 类语义角色 + Hermes 派单回写 |
| 谁负责协调多 cc？ | 没有协调者，Yjs UndoManager 按 origin 区分本地与远端，每人撤销自己的操作不影响他人 |
| 性能瓶颈？ | 单房间约可承载 8-10 人。文件超 4MB 不入云（防止 yjs 同步阻塞） |

## 🔧 故障应急

| 现象 | 立刻做 |
|------|--------|
| 项目库点回放没反应 | 先看卡片是否有「N 帧」标签 — 没有就只有 1 帧不能回放 |
| 三人头像不同步 | 刷新一次浏览器，让 awareness 重连 |
| 画布卡顿 | 节点超 100 个时打开 ⚙ → 主干视图（折叠 ROLE/反驳） |
| 回放时画布发光不消失 | 点 PlaybackScrubber 右上角「退出」按钮 |
| ali 反驳/拆解按钮没反应 | 现在已加 60s/120s 超时和失败 alert，60 秒内必有反馈 |
