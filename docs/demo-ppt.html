// 02 / PAIN POINT

当 AI 的链条变长, 黑箱就吃掉了你的判断力

我们用 LLM 做长流程协作, 每一步都是"调用 → 输出 → 下一步". 但当链路从 1 步变成 6 步、再变成多 Agent 并行时, 用户失去了对"AI 此刻在做什么 / 为什么这么做 / 哪一步出错了"的直觉.

现状的三个失能
  • 过程不可见 · Hermes / agent 流程像黑箱, 中间步骤无可视化, 只看到最终输出.
  • 结果不可反驳 · LLM 倾向于说"看起来对", 缺少对自己结论的压力测试.
  • 多人不可协作 · 单机 SaaS 工具, 不能让团队同时看到、同时改、回放别人的决策路径.

// 03 / GENESIS · 思路诞生

三人组队 48h. 把"反驳能力"和"原认知"装进画布.

5/2 上午 11:20 三人开第一次组队会. 否决了"做 4 个独立 AI 工具"的拼盘思路, 选择把"假设拆解 + 自我反驳"嵌进同一张画布.

原始 4 个 AI 模块构想
  • 假设拆解 agent · 追问基本假设, 输出假设图谱.
  • 问题重构器 · 剥离行业固有认知, 从零设计方案.
  • 自我反驳 agent · 极限挑战者, 找出未检验假设.
  • 隐含假设扫描器 · 找出领域内默认为真的习惯性假设.
最终确定优先开发"假设拆解器"和"自我反驳"两个模块 — 自我反驳可以避免开发者一厢情愿陷入自嗨, 帮助发现容易忽略的问题. 黑客松 AI 项目组队与开发准备讨论 · 2026-05-02 11:20
收敛后的产品定义

不是 4 个独立工具, 是 一张画布 + 三个决策按钮:

  • 元认知 · 输入一句话, 6 阶段在画布上自动展开. 拆解 → 角色涌现 → 执行 → 反思 → 综合.
  • 拆解 · 任意 ENTITY 节点点一下, 把它再拆成更细的本体子节点 (limit 3 层).
  • 反驳 · 任意节点点一下, 调 Devil's Advocate 生成挑战卡, 给出严重度评分 + Hermes 二次验证.
原认知系统: 执行 → 反思 → 学习 → 进化 的循环逻辑. 一个循环任务只保留 3-5 个智能体, 避免多智能体传递信息出错. 黑客松 AI 项目组队与基于 Hermes 的原认知智能体开发讨论 · 2026-05-02 10:32
// 04 / COLLISION · 架构碰撞

从"画一张图"碰撞出"做一个中台"

5/2 晚 20:09 第二次会, 4 人对齐技术架构. 决定 ALETHEIA 不只是 UI, 而是面向多个 AI 平台的调度中台 — 由它判断任务复杂度, 路由到本地 Agent / DeepSeek / Hermes.

关键决策 #1 · 任务路由策略
  • 简单任务 → 本地 DeepSeek API · 0.001 ¥/次, 成本可控.
  • 复杂多步任务 → Hermes 调度 · 利用其 skills 模块和团队 10 年训练数据.
  • 反驳校验 不挂在每一步, 只在完整方案生成后触发 — 避免 token 浪费.
关键决策 #2 · 协作底层
  • Yjs CRDT 做实时画布同步. 三人开浏览器即所见即所得, 无中心冲突.
  • 项目库写到 yjs.projects map, 三人共享同一份历史项目, 卡片显示 owner 头像.
  • 飞书机器人 作为对外沟通入口, 后续接画布作总控.
画布把每个任务拆分为多个节点, 输入需求后自动拆分节点, 每个节点自动调用对应大模型执行. 由统一决策系统控制整体逻辑. AI 项目开发团队线上会议讨论 · 2026-05-02 22:14

// 06 / BUTTONS · 三大决策按钮

画布上每个节点都长着三个"思考关节"

ALETHEIA 没有复杂菜单. 一个本体节点上, 用户能做的就三件事 — 向上元认知 · 向下拆解 · 横向反驳. 每件事都触发一条独立的 AI 链路, 把结果以子节点 / 容器形式连回画布.

⚡
META · 元认知

从一句话开出 6 阶段

输入"在上海开咖啡馆", 自动展开意图 → 拆解 → 角色涌现 → 拓扑 → 执行 → 反思. 每完成一步在项目库自动 commit 一帧, 后续可时间轴回放.

⬇
DECOMPOSE · 拆解

ENTITY 节点的"再追问"

点 ENTITY 节点上的拆解按钮, 把它再细分为子本体节点. 拆解结果挂入decomposeGroup容器, 与原节点用 dashed 边相连. 限制 3 层防失控.

⚖
CHALLENGE · 反驳

Devil's Advocate 压力测试

点任意节点的反驳按钮, 自动生成 3-5 张挑战卡 — 每张含角度、论点、严重度评分. 二次验证按钮调 Hermes 给反驳本身打可信度分.

每个节点只需要列出任务清单和关联的 Agent 或技能, 移除单个节点独立的反驳逻辑, 由统一决策系统控制整体逻辑. AI 项目开发团队线上会议讨论 · 2026-05-02 22:14
// 07 / META · 元认知按钮

⚡ 元认知 — 一句话项目的 5 步状态机

每完成一步, 自动给项目库追加一帧快照, label 用中文步骤名. 整个状态机也是时间轴回放的"最小连续帧"来源.

// 08 / DECOMPOSE · 拆解按钮

⬇ 拆解 — 把 ENTITY 节点的"再追问"装进容器

用户在 ENTITY 节点上点拆解, AI 输出 N 个子本体节点. 每次拆解都生成一个独立 decomposeGroup 容器, 子节点挂入容器, 用 dashed 边连回原节点.

// 09 / CHALLENGE · 反驳按钮

⚖ 反驳 — Devil's Advocate 给你的方案泼三盆冷水

点任意节点的反驳, 自动生成 3-5 张挑战卡 (角度 · 论点 · 严重度 · 修复 todo). 每张卡都能调 Hermes 二次验证 给反驳本身打可信度分, 防止 LLM 编造不切实际的论点.

// 10 / COLLAB · 三人共享 + 时间轴回放

三人画一张图. 后来人能"回放"前人怎么决策的.

Yjs CRDT 让三人浏览器零延迟同步同一张画布. 项目库写到 yjs projects map, 每个项目自带 commits[] 序列, 最多 30 帧. 任一人在卡片上点 ▶ 回放, 画布外圈出现暖色脉冲边框, 底部 scrubber 拖动看不同帧.

CRDT 协作能力
  • Yjs UndoManager · 按 origin 区分本地 / 远端 — 每人撤销自己的操作不影响他人.
  • Awareness 实时光标 · 拖节点时其他屏出现暖色脉冲气泡.
  • Project owner 字段 · 卡片显示 owner 圆点 + 用户名, 顶部"按用户筛选"chip.
  • 离线后回来 · CRDT 自动合并, 无中心冲突.
PlaybackScrubber · 双模式回放
  • 按项目 · 顺序播 commits[] 全部帧. 看完整决策链路.
  • 按用户 · filter 出某个 owner 的活动帧 — 看"小叶子今天在画布上做了什么".
  • scrubber · range slider 拖到任意帧, 画布即刻切到那一刻状态. 1.5s 一帧自动播放.
  • 视觉锁 · 回放时画布外圈暖色脉冲发光 + 顶部"PLAYBACK · 回放模式"标签, 编辑全 noop.
领导打开画布即可查看每个节点当日完成情况, 系统自动识别不同工种衔接的冲突, 还能给出进度建议… 不需要频繁开进度会. AI 项目开发团队线上会议讨论 · 2026-05-02 22:14

// 12 / DEBUG · 修 bug 全流程

案例: "切主干模式画布崩了" · 三轮修复才彻底

React Flow 11 在 child layout 时 throw Parent node X not found, 不走 onError 通道, 直接冒泡到 ErrorBoundary 把整画布崩掉. 反复迭代三轮才解决.

v1 — 清 parentNode 把孤儿 child 的 parentNode 字段直接清掉 结果: 子节点变绝对坐标全堆左上角. 用户图 41 反馈"反驳还有排版重新进去后都没有了". 弃用.
v2 — filter 孤儿 直接从 nodes 数组过滤掉孤儿子节点 结果: PDF 等顶层节点偶尔被误判 parent 也消失. 用户图 42 反馈"包括 pdf 不见了". 弃用.
v3 — fallback root + onError 002 挂个隐形 8000×8000 group 兜住所有孤儿 解决了"孤儿子节点"场景. 但用户切"主干"模式时 — group 自身被 hidden, 子节点没 cascade hide — React Flow 仍 throw, 这次还冒到了 ErrorBoundary. 部分解决.
v4 — cascade hidden + filter (最终) 父不可见, 强制子也不可见, 然后整体 filter cascade 3 pass 兜底 (projectGroup → challengeGroup → leaf). 最终交给 React Flow 的 nodes 数组里所有 parentNode 引用都自洽. commit 49041d5 上线.
测试在线 demo 时, 误进入本地环境, 进入后无法正常退出, 当前不知道退出方法. 青白江开啤酒馆计划与 AI 决策系统测试闲聊记录 · 2026-05-03 18:14
