具体发现

CR-001 MAJOR Type: Design Impact · File: src/pages/KnowledgeGraph.jsx

Evidence：已 745 行（pre-existing 超限）。本次又新增 ~75 行 yjs 启动 useEffect、RemoteUserList、AI 设置按钮、N 键快捷键。useEffect 依赖列表已经 8 项以上。

Required update：抽出 useCollabSession(room, name, color) hook 封装 startSync + attachYjsSync + setLocalUser + 卸载清理；抽出 <CollabHeader/> 子组件包装"在线用户列表 + 房间号 + AI 设置 + 退出按钮"四件套。目标：KnowledgeGraph ≤ 500 行。

CR-002 MINOR Type: Ownership · File: src/collab/yjsSync.js

Evidence：第 162 行 useCanvasStore.setState({ nodes, edges }) 直接写 state，绕过 store 已有的 setNodes/setEdges action。

Required update：改为 const s = useCanvasStore.getState(); s.setNodes(nodes); s.setEdges(edges)。让 store 是状态自治的权威边界，sync 层只调 actions。

CR-003 MINOR Type: Duplication · File: src/components/canvas/NodePropertyPanel.jsx

Evidence：新加的通用 section 已经包含色板 + color picker；但 categoryNode 的 case 仍保留独立 color 字段（行 ~196）。

Required update：移除 categoryNode case 内的 color 字段，让通用 section 统一处理；categoryNode 只保留"分类名称"字段。

CR-004 MINOR Type: ValidationGap · File: e2e/multi-user-collab.spec.js

Evidence：测试覆盖核心 sync 但缺断线重连场景。Yjs 自带重连，但没测过"A 断网→改东西→重连后合并"。

Required update：加 P1 测试：A page 用 page.context().setOffline(true) 离线 → 改节点 → 上线 → 验证最终一致。

CR-005 MINOR Type: Complexity · File: src/collab/yjsSync.js

Evidence：启动序列（暂存 → 清空 → 等 sync → 决定推/拉 → 解锁）用模块级 let 状态变量管理，可读性中等。

Required update：包装为 class SyncEngine，启动状态变成显式枚举（IDLE / WAITING_SYNC / SYNCED）。提升可测性 + 可同时管多个 doc。

CR-006 MINOR Type: Cleanup · File: src/services/aiService.js

Evidence：客户端规则解析（extractConcepts / suggestRelations）和 LLM 增强（analyzeWithLLM / suggestRelationsLLM）并存，目前 BottomAIBar 仍用规则解析。两套并存合理（mock fallback），但调用约定未在文档说明。

Required update：在 aiService.js 顶部注释中明确"何时用 extractConcepts vs analyzeWithLLM"的决策树；或者让 BottomAIBar 显式选 mode。

值得保持的设计亮点

  • Bounded Local Spine 显式建模：finalizeStartup 启动序列没有用 setTimeout 黑魔法堆出来，而是显式以"暂存 → 等 sync → 决定推拉 → 解锁"四步描述，并保留扩展点
  • 三轨 AI Provider 工厂：claude-cli-bridge（默认零成本）/ openai-like（通用兜底）/ mock（无网络）三条平行通路统一在 callLLM 之下，新增 provider 只需要加 PROVIDER_PRESETS 一条
  • 本机 claude CLI 桥的安全设计：默认 HOST=127.0.0.1 物理隔离 + CORS 白名单 + 强调"只能本机跑"的注释 = 三道闸阻止公网调用滥用
  • 每用户独立选中状态：nodeToPlain 故意 skip selected 字段，让本地选中独立同步——这是协作画布的常见经典处理，避免互相干扰
  • 渐进式协作落地：useCanvasStore 没改 actions 接口，攻入点是 yjsSync 这个外挂模块；不开协作就是单机模式，向后完全兼容
  • 测试稳定性策略：E2E 用 window.__canvasStore 调 store API 而非 dblclick 等 UI 事件，绕过浏览器时序差异让测试可重复
  • 建筑极简风格一以贯之：JoinRoom 入口页 + AiSettingsPanel + RemoteUserList 视觉与现有画布风格无缝衔接（暖色 #c8a882 点缀 + Noto Serif/Sans + 1px 细线）
  • Caddy 部署文档化：deploy/ 目录三件套（deploy.sh + Caddyfile.canvas + know-canvas-yws.service）+ README 形成完整可重复部署流
